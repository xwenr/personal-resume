## 附 · GitHub Token 配置（Private 仓库专用）

> 你的 `resumeflow` 和 `travel-plan` 是 Private 仓库，匿名 API 访问不到。按下面五步搞定：

### 一、去 GitHub 生成 Token

1. 登录 GitHub，打开 <https://github.com/settings/personal-access-tokens>
2. 右上角点击 **Generate new token** → 选择 **Fine-grained token**（不是 Classic）
3. 按下面填：

| 字段 | 推荐值 |
|---|---|
| **Token name** | `velorah-site-build` |
| **Expiration** | `90 days`（到期再生成新的即可） |
| **Repository access** | `Only select repositories` → **勾选 `resumeflow` 和 `travel-plan`** |
| **Repository permissions** → **Metadata** | `Read-only` ✅ _必选_ |
| **Repository permissions** → **Contents** | `Read-only` ✅ _预留，后面看 README 会用到_ |

4. 点最下面的 **Generate token**，复制出现的 `github_pat_…` 字符串（只显示一次！）

### 二、在项目里配置

在项目根目录（`velorah/`）下，**新建一个文件叫 `.env.local`**（不要提交到 Git，`.gitignore` 里已经 `*.local` 兜底）。

内容照抄：

```bash
GITHUB_TOKEN=github_pat_REPLACE_ME_WITH_YOUR_TOKEN
```

把 `github_pat_REPLACE_ME_WITH_YOUR_TOKEN` 替换为你刚刚复制的 Token。

> 💡 也有现成模板 `.env.example`，直接 `cp .env.example .env.local`（Windows PowerShell: `Copy-Item .env.example .env.local`），把 Token 填进去即可。

### 三、拉一次数据验证

```powershell
npm run fetch:github
```

预期输出：

```
[fetch-github] 4 repos: xwenr/resumeflow, xwenr/travel-plan, xwenr/promptory, xwenr/Fruit-Tea-Liquor
[fetch-github] using GITHUB_TOKEN (ghp_…abcd) — 5000/hr, can read private repos.
  ✓ xwenr/resumeflow    lang=…  ★0   updated=…
  ✓ xwenr/travel-plan   lang=…  ★0   updated=…
  ✓ xwenr/promptory     lang=TypeScript  ★0   updated=2026-04-23
  ✓ xwenr/Fruit-Tea-Liquor lang=JavaScript  ★0   updated=2026-04-17
[fetch-github] wrote ./src/data/github-repos.generated.ts
```

如果看到 4 个 ✓ 就说明成功了。打开 <http://localhost:5174/> 刷新页面即可看到新卡片。

### 四、数据可见性（重要）

> ⚠️ 配了 Token 后，**仓库里的代码不会外泄**，但**你主动展示的元数据（仓库名 / description / 主语言 / Stars / 最近更新时间）会进入网站的静态 HTML，任何访客都能看到**。
>
> 对一个"作品集"网站来说这是**预期行为**——你主动要让 HR 看到这些项目。但如果某个 Private 仓库你不想被看到，**就别放进 `works.config.ts`**。
>
> Token 本身**只**存在于你的本地 / CI 构建机器上，**绝不会出现在打包产物里**。

### 五、生产环境部署

把站点部署到 Vercel / Netlify / Cloudflare Pages 时：

1. 在平台的 **Environment Variables** 设置里加一条：
   - Key: `GITHUB_TOKEN`
   - Value: 同上的 `github_pat_…`
2. 构建命令保持 `npm run build`（内部会自动执行 `npm run fetch:github`）
3. 部署时脚本会拿 Token 去 GitHub 拉最新元数据，然后打包成静态站点

**Token 过期了怎么办？** 去 GitHub 设置里重新生成一个同名 Token（会自动替换旧的），改 `.env.local` / 平台 env 里的值即可。站点不会自动刷新数据——需要手动重新 build 或者配 GitHub Actions 每日自动构建。
